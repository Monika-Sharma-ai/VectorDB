What is Nearest Neighbor Search?

Find the closest vectors to a query in some embedding space.

Definition Given a query Q, return the data points closest to it under some distance (L2, cosine, inner product).

  • Image search — find similar photos
  • Recommenders — items close to user vector
  • RAG / semantic search — passages near the question
  • Deduplication — near-identical content

Exact NN compares Q against every point — accurate but O(N · d) per query.

Why Approximate?

Trade a tiny bit of accuracy for massive speed and memory savings.

At 1B vectors × 768 dims, brute-force is impossible in real time.

  • Exact — scans every point. 100% recall, slow.
  • Approximate — searches a small region. ~95–99% recall, 1000× faster.

You pick the operating point on the recall ↔ speed ↔ memory triangle.

efSearch ↑ recall ↑ latency ↑

The Curse of Dimensionality

In high dimensions, distances collapse — every point looks "equally far".

As d grows, the volume of a unit ball concentrates near its surface; nearly all data sits at the edge.

  • Distances d_min ≈ d_max → ranking becomes unstable
  • Tree splits prune almost nothing past ~20 dims
  • Brute force becomes the only exact option — and it's too slow

Implication: we need graphs (HNSW) or quantization (IVF + PQ) — not classical trees — for modern embeddings (d = 384–1536).
