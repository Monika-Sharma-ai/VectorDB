Locality-Sensitive Hashing (LSH)

Design hash functions where similar vectors collide in the same bucket — on purpose.

Intuition Normal hashes scatter inputs randomly. LSH is built so close points share a hash; far points don't.

Step 1Draw K random hyperplanes
Step 2Bit_i = sign(h_i · v) → K-bit code
Step 3Store v in bucket "code"
Step 4Hash Q → scan its bucket only

Use L hash tables in parallel → boost recall (a candidate hits any of them).

Random Projection — Compress First, Search Faster

Multiply by a random matrix → tiny vectors that approximately preserve distances (Johnson–Lindenstrauss).

Intuition A randomly chosen low-dim subspace keeps pairwise distances within a small ε with high probability.

  • Input: v ∈ ℝ⁷⁶⁸ → Random matrix R ∈ ℝ³²ˣ⁷⁶⁸ → Output: v' ∈ ℝ³²
  • ‖v'_a − v'_b‖ ≈ ‖v_a − v_b‖ (with small distortion)
  • Used as a preprocessing step before tree / hash / quantization

Why it works: in high dim, almost every random direction is "informative" — dimensions act independently.

SimHash — Binary Fingerprints for Cosine Similarity

K random hyperplanes → K-bit fingerprint. Compare with Hamming distance (XOR + popcount).

Intuition Each hyperplane votes "left or right" → one bit. Stack K bits to get a fingerprint where similar vectors share most bits.

  • bit_i = sign(h_i · v) → 0 or 1
  • Hamming(a, b) = number of differing bits ≈ angle between a, b
  • Compares millions of fingerprints in milliseconds (CPU XOR + popcount)

Famous use: Google's near-duplicate web page detection at web scale.
