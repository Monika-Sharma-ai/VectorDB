🚀 Qdrant Explained

The blazing-fast, production-grade vector search engine — explained for students.

Written in Rust HNSW Filtering Production-Ready

1. What is Qdrant?

Qdrant (pronounced "quadrant") is an open-source vector search engine and database written in Rust. It's designed for production scale — millions to billions of vectors — with strong support for filtering on metadata (called "payload" in Qdrant).

🚀 Analogy — The Smart Airport Control Tower:
Picture a busy airport. Thousands of planes (vectors) are in the sky. Each plane carries cargo (payload: airline, destination, weight). When someone asks "find me the 5 nearest cargo planes flying to Europe with weight > 10 tons," the control tower:
  1. Filters by destination & weight first,
  2. Then finds the geographically closest planes — instantly.
Qdrant is that tower: it combines vector similarity + structured filtering at lightning speed.

2. Qdrant's Core Concepts

📦 Collection

A named container of vectors (like a "table"). You define its dimension & distance metric.

🎯 Point

A single record = id + vector + payload (metadata JSON).

🏷️ Payload

Any JSON metadata: tags, prices, categories, dates — fully filterable.

📐 Distance

How similarity is measured: Cosine, Dot, or Euclid.
POST /collections/movies
{
  "vectors": { "size": 384, "distance": "Cosine" }
}

POST /collections/movies/points
{
  "points": [
    { "id": 1, "vector": [0.12, -0.44, ...],
      "payload": { "title": "Inception", "year": 2010, "genre": "sci-fi" } },
    { "id": 2, "vector": [0.21, -0.31, ...],
      "payload": { "title": "Titanic",   "year": 1997, "genre": "romance" } }
  ]
}

3. How Qdrant Stores Vectors — The HNSW Graph

Qdrant's main index is HNSW (Hierarchical Navigable Small World). Imagine a multi-level subway map:

🚇 Analogy — Subway with Express Lines:
Bottom layer = every single station. Middle layer = regional trains stopping at fewer stations. Top layer = bullet trains hitting just a few hubs.
To get to a far destination, you start with the express, jump down to regional, then to local — covering huge distances in just a few hops. That's exactly how HNSW search travels through vector space.

4. Qdrant's Superpower: Filtered Vector Search

Most vector DBs filter after finding nearest neighbors (post-filtering), which can miss results. Qdrant uses filterable HNSW — it filters during the graph walk, guaranteeing accurate results even with strict filters.

POST /collections/movies/points/search
{
  "vector": [0.12, -0.44, ...],
  "limit": 5,
  "filter": {
    "must": [
      { "key": "genre", "match": { "value": "sci-fi" } },
      { "key": "year",  "range": { "gte": 2000 } }
    ]
  }
}
🛒 Analogy — Smart Shopping Mall Directory:
You ask: "Show me stores similar to Apple, but only ones on the 2nd floor that are open after 9 PM." A normal directory would list nearest stores, then filter — and might return zero. Qdrant walks the directory while applying filters at every step, so you always get the best matching open stores.

5. How Searching Actually Happens

1
You send a query vector (and optional filter) to Qdrant.
2
Entry point on the top HNSW layer is selected.
3
Greedy hop: at each node, jump to the neighbor closest to q (that also matches the filter).
4
Drop a layer when no closer neighbor exists; repeat hopping in finer detail.
5
At layer 0, collect top-k nearest points and return their IDs, scores, and payload.
🗺️ Analogy — Treasure Hunt with a Helicopter:
Start in a helicopter (top layer) for a wide view, spot the right region, descend to a car (middle layer) to drive to the right neighborhood, then walk on foot (bottom layer) to find the exact treasure. Each step gets more precise.

7. Real Python Example

from qdrant_client import QdrantClient
from qdrant_client.models import VectorParams, Distance, PointStruct, Filter, FieldCondition, MatchValue, Range

client = QdrantClient(url="http://localhost:6333")

# 1. Create a collection
client.recreate_collection(
    collection_name="movies",
    vectors_config=VectorParams(size=384, distance=Distance.COSINE),
)

# 2. Insert points (vector + payload)
client.upsert(
    collection_name="movies",
    points=[
        PointStruct(id=1, vector=[...], payload={"title":"Inception","year":2010,"genre":"sci-fi"}),
        PointStruct(id=2, vector=[...], payload={"title":"Titanic",  "year":1997,"genre":"romance"}),
    ],
)

# 3. Search with filter
results = client.search(
    collection_name="movies",
    query_vector=[...],
    limit=5,
    query_filter=Filter(must=[
        FieldCondition(key="genre", match=MatchValue(value="sci-fi")),
        FieldCondition(key="year",  range=Range(gte=2000)),
    ]),
)
for r in results:
    print(r.id, r.score, r.payload)
